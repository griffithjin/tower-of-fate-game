# 核心玩法数学验证

## 牌库系统验证

### 基础数据
```
初始牌库：4花色 × 13点数 × 4副本 = 208张
每层消耗：4守卫亮牌 + 4玩家出牌 = 8张
13层消耗：8 × 13 = 104张
剩余牌库：208 - 104 = 104张
```

### ✅ 验证通过
每层消耗计算正确。

## 概率计算验证

### 上升概率（点数匹配）
```
基准概率（随机选）：1/13 ≈ 7.69%
最优策略（选剩余最多的点数）：约8.0-8.5%
```

### 完美匹配概率（点数+花色）
```
P = 该花色该点数的剩余张数 / 总剩余张数

初始：4/208 = 1/52 ≈ 1.923%
第N层后：(4 - 消耗数) / (208 - 8N)

关键发现：四副本设定下，完美匹配概率并非完全恒定
实际变化：从1.923%逐渐变化到约1.9-2.0%

近似恒定，对游戏设计影响可接受
```

### ❌ 发现问题：抽卡获取率过低
```
13层期望获得抽卡次数 = 13 × 1.923% ≈ 0.25张

这意味着平均每4局才能获得1张天命卡！
```

**建议调整：**
1. 增加"连击奖励" - 连续上升3层获得1张抽卡
2. 增加"层数奖励" - 达到第7层自动获得1张抽卡
3. 增加"完美防御" - 对手上升时你匹配点数，获得抽卡
