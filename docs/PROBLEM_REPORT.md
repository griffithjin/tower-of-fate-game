# ⚠️ 新机制问题报告

## 发现的核心问题

### 问题1：牌库不足以支撑13层

**计算：**
- 总牌库：52张
- 每层消耗：
  - 守卫牌：1张
  - 4位玩家出牌：4张
  - **合计：5张/层**
- **最大可持续层数：52 ÷ 5 = 10.4层**

**结论：** 在第11层牌库就会耗尽，**无法完成13层！**

### 问题2：手牌消耗过快

- 玩家初始5张手牌
- 每层出1张
- 5层后手牌耗尽
- 即使补充，整体牌库仍然不够

---

## 🎯 解决方案（金蛇建议）

### 方案A：增大牌库（推荐）

**修改：**
- 守卫牌库：104张（2副牌）
- 玩家手牌：从104张中抽取
- 每层消耗5张，可支持 **20层** > 13层 ✅

**优点：** 保持现有机制，简单直接
**缺点：** 完美匹配概率降低（1/104）

### 方案B：循环牌库

**修改：**
- 玩家打出的牌回收至牌库底
- 牌库循环使用
- 守卫牌不回收（或也回收）

**优点：** 可持续无限层
**缺点：** 失去"记忆博弈"的核心乐趣

### 方案C：减少每层消耗

**修改：**
- 只有成功匹配的玩家才消耗手牌
- 失败的玩家不消耗（或回收）
- 每层消耗：1 + (0~4) = 1-5张

**优点：** 节省牌库
**缺点：** 改变游戏节奏

### 方案D：动态补充

**修改：**
- 每层结束后，所有玩家弃掉剩余手牌
- 重新从牌库抽取5张
- 弃牌回收或移除

**优点：** 每局都是新决策
**缺点：** 失去"手牌规划"深度
