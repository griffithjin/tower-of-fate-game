## 仪表板概览

### 实时统计
管理仪表板提供全面的实时数据：

- **活跃玩家**：当前在线用户
- **进行中的游戏**：活跃游戏数量
- **今日收入**：来自所有来源的日收入
- **新注册**：今日注册用户
- **服务器状态**：所有游戏服务器的健康状况

### 导航
```
仪表板
├── 概览
├── 玩家
├── 游戏
├── 锦标赛
├── 资产
├── 分析
├── 设置
└── 支持
```

### 关键指标卡片
1. **玩家参与度**
   - DAU（日活跃用户）
   - MAU（月活跃用户）
   - 平均会话时长
   - 留存率（次日、7日、30日）

2. **财务指标**
   - 日收入
   - ARPDAU（每活跃用户平均收入）
   - 转化率
   - 高消费用户

3. **游戏健康度**
   - 匹配成功率
   - 服务器响应时间
   - 错误率
   - 支持工单量

## 资产管理

### 塔楼模型

#### 上传新塔楼
1. 导航到 资产 → 塔楼 → 添加新塔
2. 填写塔楼详情：
   - **名称**：塔楼显示名称
   - **主题**：选择主题类别
   - **难度**：简单 / 普通 / 困难
   - **地区**：可用地区
3. 上传塔楼图片：
   - 预览图 (512x512px)
   - 完整视图 (1024x1024px)
   - 3D模型文件 (.glb, .gltf)
4. 配置3D设置：
   - 旋转速度
   - 动画触发器
   - 灯光效果
5. 设置难度参数：
   - 怒气卡频率
   - 特殊卡生成率
   - 奖励倍数
6. 保存并发布

#### 编辑塔楼配置
```javascript
// 塔楼配置示例
towerConfig = {
  id: "tower_001",
  name: "神秘城堡",
  theme: "fantasy",
  difficulty: "hard",
  floors: 13,
  angerCardRate: 0.15,      // 15% 概率
  specialCardRate: 0.08,    // 8% 概率
  rewardMultiplier: 1.5,
  regions: ["CN", "US", "EU", "ASIA"]
}
```

### 卡牌组

#### 创建新卡组主题
1. 转到 资产 → 卡牌 → 卡组主题
2. 点击"创建主题"
3. 配置：
   - 主题名称
   - 卡牌背面设计
   - 配色方案
   - 特殊效果
4. 设计卡牌正面：
   - 数字牌 (2-10)
   - 面牌 (J, Q, K, A)
   - 特殊卡牌
5. 设置卡牌能力：
   - 效果类型
   - 触发条件
   - 持续时间
   - 冷却时间
6. 在模拟器中测试卡组
7. 发布给玩家

#### 编辑卡牌符号
```css
/* 卡牌符号样式 */
.card-spade { color: #000000; }
.card-heart { color: #ff0000; }
.card-diamond { color: #ff0000; }
.card-club { color: #000000; }
.card-special { 
  background: linear-gradient(45deg, #gold, #yellow);
  border: 2px solid #orange;
}
```

#### 配置特殊效果
| 效果类型 | 描述 | 配置 |
|---------|------|------|
| 护盾 | 阻挡怒气卡 | 持续时间：1回合 |
| 传送 | 跳跃楼层 | 范围：任意楼层 |
| 交换 | 交换位置 | 目标：单个玩家 |
| 双倍 | 下一次匹配双倍 | 倍数：2倍 |

### 明信片

#### 上传明信片图片
1. 导航到 资产 → 明信片
2. 点击"上传新图片"
3. 上传图片：
   - 全分辨率 (1200x800px)
   - 缩略图 (300x200px)
4. 添加元数据：
   - 标题
   - 祝福消息
   - 稀有度等级
   - 可用期限
5. 设置地区限制（可选）
6. 发布

#### 编辑祝福消息
```json
{
  "postcard_id": "pc_001",
  "title": "好运",
  "message": "愿卡牌眷顾你！",
  "translations": {
    "en": "May the cards be in your favor!",
    "ja": "カードがあなたに味方しますように！",
    "ko": "카드가 당신의 편이 되길!"
  },
  "rarity": "epic",
  "send_limit": 3
}
```

#### 设置稀有度等级
- **普通**：60% 掉落率
- **不凡**：25% 掉落率
- **稀有**：10% 掉落率
- **史诗**：4% 掉落率
- **传说**：1% 掉落率

## 用户管理

### 查看用户资料
1. 转到 用户 → 玩家列表
2. 按以下条件搜索：
   - 用户名
   - 邮箱
   - 玩家ID
   - 注册日期
3. 查看详细资料：
   - 账户信息
   - 游戏统计
   - 购买历史
   - 支持工单
   - 登录历史

### 管理封禁

#### 临时封禁
1. 在玩家列表中找到用户
2. 点击"操作" → "封禁"
3. 选择持续时间：1天 / 7天 / 30天
4. 输入原因：
   - 作弊
   - 骚扰
   - 利用漏洞
   - 其他（请说明）
5. 确认封禁

#### 永久封禁
```javascript
// 封禁用户 API
POST /api/admin/users/{userId}/ban
{
  "type": "permanent",
  "reason": "cheating",
  "evidence": "replay_id_12345",
  "notes": "在锦标赛中使用加速外挂"
}
```

#### 申诉流程
1. 用户通过支持工单提交申诉
2. 管理员审查案例
3. 选项：
   - 维持封禁
   - 减少封禁时长
   - 解除封禁
4. 通知用户决定

### 处理退款

#### 审查退款请求
1. 转到 支持 → 退款
2. 查看购买详情：
   - 交易ID
   - 金额
   - 日期
   - 购买项目
3. 检查用户历史：
   - 之前的退款
   - 账户状态
   - 购买后的游戏时间
4. 决策选项：
   - 批准全额退款
   - 批准部分退款
   - 拒绝退款
5. 通过支付提供商处理退款

## 锦标赛管理

### 创建新锦标赛
1. 导航到 锦标赛 → 创建
2. 配置基本信息：
   - 锦标赛名称
   - 开始/结束日期
   - 报名期
   - 报名费
3. 设置赛制：
   - 单败淘汰
   - 双败淘汰
   - 循环赛
   - 瑞士制
4. 定义奖池：
   - 第1名：40%
   - 第2名：25%
   - 第3名：15%
   - 第4-8名：20%（分配）
5. 设置参赛资格：
   - 最低段位
   - 地区限制
   - 年龄要求
6. 发布锦标赛

### 监控锦标赛进度
```
锦标赛仪表板
├── 报名统计
│   ├── 总报名人数
│   ├── 容量填充百分比
│   └── 报名截止
├── 当前轮次
│   ├── 进行中的比赛
│   ├── 完成的比赛
│   └── 争议结果
├── 排行榜
│   ├── 当前排名
│   ├── 最佳表现者
│   └── 淘汰玩家
└── 奖金分配
    ├── 确认的获胜者
    ├── 待验证
    └── 已支付金额
```

### 处理争议
1. 审查争议索赔
2. 检查游戏回放
3. 查阅服务器日志
4. 做出裁决：
   - 维持结果
   - 反转结果
   - 需要重赛
   - 两名玩家均取消资格
5. 记录决定
6. 通知玩家

## AB测试

### 创建新测试
1. 导航到 实验 → 创建测试
2. 定义测试参数：
   - 测试名称
   - 假设
   - 主要指标
   - 成功标准
3. 创建变体：
   - 对照组 (A)
   - 变体 (B)
   - 额外变体（可选）
4. 设置流量分配：
   - 50/50 分配
   - 或自定义百分比
5. 定义受众：
   - 所有用户
   - 仅新用户
   - 特定地区
6. 设置持续时间
7. 启动测试

### 监控结果
```
AB测试："新教程流程"
├── 对照组 (A)
│   ├── 用户数：5,000
│   ├── 转化率：32%
│   └── 收入：¥8,400
├── 变体 (B)
│   ├── 用户数：5,000
│   ├── 转化率：38%
│   └── 收入：¥10,150
├── 置信度：95%
├── 获胜者：变体 B
└── 建议：部署 B
```

### 应用获胜变体
1. 审查最终结果
2. 验证统计显著性
3. 点击"应用获胜者"
4. 向100%用户推广
5. 记录学习心得

## 故障排除

### 常见问题

#### 服务器负载高
1. 检查活跃游戏数量
2. 查看玩家分布
3. 扩展服务器实例
4. 如有需要启用维护模式

#### 支付失败
1. 检查支付提供商状态
2. 验证API凭证
3. 查看错误日志
4. 联系支付提供商支持

#### 匹配问题
1. 检查玩家池大小
2. 查看匹配参数
3. 调整ELO范围
4. 监控队列时间

### 紧急程序

#### 服务器宕机
1. 启用维护页面
2. 通过社交媒体通知玩家
3. 调查根本原因
4. 应用修复
5. 逐步重启
6. 密切监控

#### 安全漏洞
1. 启用锁定模式
2. 强制重置密码
3. 审计访问日志
4. 通知受影响用户
5. 召集安全团队
6. 事后审查
